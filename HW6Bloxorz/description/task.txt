Bloxorz Game

Задача: реализовать игру Bloxorz.
 Правила приведены в description/gameRules.txt.

Задание состоит из трех частей:
 1. реализация игры в простейшем случае (без дополнительных элементов);
 2. поиск решения (в тестах доски небольшие, поэтому достаточно реализовать перебор вариантов обычным поиском в ширину);
 3. поддержка дополнительных элементов (мостов, легковесных клеток) на поле.
Чтение доски в правильном формате тоже нужно реализовать.

Рекоммендуется (но не обязательно) выполнять задание поэтапно,
 т.е. в начале сделать часть 1, потом 2, и только потом добавлять реализацию дополнительных элементов.

Для каждой из частей даны тесты, которые в итоге должны заработать:
 src/test/game/_1player
 src/test/game/_2solver
 src/test/game/_3bridges
Чтобы удостовериться, что очередная часть сделана, просто запустите все тесты в данной папке.

В папке levels собраны примеры уровней:
 levels/bloxorz - уровни исходной игры
 levels/basic - уровни для части #1
 levels/bridges - уровни для части #3
Поиск решения должен работать для всех уровней.

Вам нужно реализовать функцию createGame.
 Обратите внимание, что второй параметр нужен только для части #3, до этого он передается как null.
 Также для 3й части нужно реализовать функцию bridgesInfo,
 позволяющую декларативно объявлять структуру мостов (примеры в bridgeInfoForSampleLevels.kt).


В playGame.kt можно увидеть вашу реализацию в игре :).
 По нажатию клавиши enter за вас делается следующий правильный ход (когда вы уже реализовали поиск решения).

Ваше решение должно жить в правильно переименнованном пакете _SurnameName (и отправить нужно только эту заархивированную папку).


Некоторые замечания заранее:

- Не используйте 'Char' для того, чтобы различать типы клеток, храните данные явно.
(Такой подход используется в Game, но это сделано намеренно, чтобы не реализовывать заранее все структуры данных).

- Создайте отдельный неизменяемый класс для хранения доски и элементов на ней.
Состояния игры на доске (блок и состояния мостов) храните отдельно.
